# 발표 로드맵 (10분)


---

## 0. 전체 시간 배분
- PPT: 6분 기준
- 시연: 4분
- 합계: 9~10분

---

## 1. PPT 파트 (6분)

### 1) 한 줄 가치 제안 (0:00~0:40)
- 핵심 메시지: “이 앱은 서울의 범죄 위험을 **지금** 보여주는 지도다.”
- 말하기 포인트:
  - “사람들은 지금 어디가 위험한지 바로 알고 싶다.”
  - “우리는 통계 + 실시간 데이터를 합쳐서 한 화면에 보여준다.”

### 2) 문제와 기존 한계 (0:40~1:30)
- 핵심 메시지: “기존 통계는 과거에 머물고, 장소 체감이 어렵다.”
- 말하기 포인트:
  - “CSV 통계는 어렵고, 지금 상황을 말해주지 않는다.”
  - “지도와 혼잡도가 없으면 ‘어디가 위험한지’ 직관이 없다.”

### 3) 해결 방식 (IoT 관점) (1:30~2:40)
- 핵심 메시지: “이 앱은 ‘도시 IoT 신호(혼잡도) + 범죄 통계’를 합쳐 위험도를 만든다.”
- 말하기 포인트:
  - “도시는 센서/데이터로 움직인다 → 사람 흐름이 IoT처럼 감지된다.”
  - “서울시 혼잡도 API는 실시간 도시 센서 신호 역할을 한다.”
  - “경찰 통계는 ‘과거 패턴’, IoT는 ‘현재 상태’를 알려준다.”
  - “둘을 합치면 ‘지금 위험할 가능성’을 더 정확하게 보여준다.”

### 4) 핵심 기능 3가지 (2:40~4:30)
- 핵심 메시지: “3가지 화면으로 문제를 바로 해결한다.”
- 기능 요약:
  1. 시간대 분석: 언제 위험한지 (시간 페이지)
  2. 요일 분석: 어떤 요일이 위험한지 (요일 페이지)
  3. 지역/핫스팟: 어디가 위험한지 (지역/핫스팟 페이지)

### 5) 투자/소개 포인트 (4:30~5:40)
- 핵심 메시지: “확장 가능하고, 데이터 기반 의사결정에 바로 쓸 수 있다.”
- 말하기 포인트:
  - “다른 도시/데이터로 확장 가능”
  - “공공기관/치안/상권 분석에 활용 가능”
  - “자동 수집 + 대시보드로 운영 비용이 낮다”
  - “IoT 신호(혼잡도/유동인구)만 붙이면 실시간 서비스로 확장 가능”

---

## 1-1. PPT 키워드와 시각자료 가이드

### 키워드(슬라이드 제목에 쓰기 좋은 단어)
- 실시간 위험도
- 도시 IoT 신호
- 혼잡도 = 현재 상태
- 범죄 통계 = 과거 패턴
- 결합(퓨전)
- 지도에서 바로 이해
- 확장 가능 / 자동 수집

### 시각자료(슬라이드에 들어갈 그림/차트 아이디어)
- 문제 슬라이드:
  - “CSV 표 vs 지도” 비교 이미지 (직관성 강조)
  - “과거 데이터만 보는 한계” 아이콘(시계 + 느낌표)
- 해결 슬라이드(IoT 연결 강조):
  - “IoT 센서 → 실시간 혼잡도 → 위험도” 흐름 다이어그램
  - “과거(범죄 통계) + 현재(혼잡도)”가 합쳐지는 Venn diagram
- 기능 슬라이드:
  - 시간/요일/지역 3분할 스크린샷 미리보기
- 투자/확장 슬라이드:
  - “도시 확장 지도(서울 → 타 도시)” 화살표 이미지
  - “유동인구/날씨/행사 데이터 추가” 레이어 그림

## 2. 시연 파트 (4분)

### 1) 홈 화면 (0:00~0:40)
- 보여줄 것: 앱 전체 구조, 메뉴
- 한 줄 설명: “이 앱은 ‘언제/어디가 위험한지’를 바로 찾게 해줍니다.”

### 2) 시간 페이지 (0:40~1:40)
- 보여줄 것: 시간대별 위험 추이 그래프 + 핵심 지표
- 한 줄 설명: “밤/새벽에 위험도가 올라가는 걸 눈으로 확인할 수 있습니다.”

### 3) 요일 페이지 (1:40~2:40)
- 보여줄 것: 요일별 비교 그래프
- 한 줄 설명: “어떤 요일이 위험한지 한눈에 비교됩니다.”

### 4) 핫스팟 페이지 (2:40~4:00)
- 보여줄 것: 지도 + 위험 지역 표시
- 한 줄 설명: “지금 사람 많은 곳 + 위험도가 높은 곳을 바로 보여줍니다.”
- 마무리: “이렇게 실시간으로 위험도를 보여주는 것이 우리 앱의 핵심입니다.”

---

## 3. 한 줄 결론
- “이 앱은 ‘지금 위험한 곳’을 누구나 쉽게 이해하게 만드는 실시간 범죄 위험 지도입니다.”

## 4. 6분 발표용 PPT 작업 지시서 (8장 기준)

아래는 “PPT 제작 담당자”에게 전달할 수 있도록 구체적으로 작성한 지시서입니다.  
슬라이드 수, 메시지, 시각자료, 자막 문구까지 포함합니다.

### 전체 규칙
- 총 8장, 발표 파트 6분 기준.
- 슬라이드 1장당 40~45초 분량.
- 문장은 중학생도 이해할 수 있는 쉬운 단어 사용.
- 각 장의 제목은 6~10자, 한 줄 메시지는 1문장만.

### 슬라이드별 작업 지시
1. 타이틀/가치 제안
   - 제목: “실시간 범죄 위험 지도”
   - 한 줄 메시지: “지금 위험한 곳을 누구나 바로 보게 만드는 지도”
   - 시각자료: 메인 화면 스크린샷 1장(흐림 처리 가능)
   - 배치: 큰 제목 상단, 메시지 중앙, 스크린샷 하단

2. 문제(왜 필요한가)
   - 제목: “기존 통계의 한계”
   - 한 줄 메시지: “과거 통계만 보면 지금 위험을 알기 어렵다”
   - 시각자료: CSV 표 vs 지도 비교 이미지
   - 포인트 아이콘: 시계 + 느낌표

3. 해결 방식(IoT 관점)
   - 제목: “통계 + IoT 현재 신호”
   - 한 줄 메시지: “과거 패턴과 현재 상황을 합쳐 위험도를 만든다”
   - 시각자료: Venn diagram(범죄 통계 / 실시간 혼잡도)
   - 추가: “혼잡도 = 도시 IoT 신호” 문구 작게 표기

4. 데이터 파이프라인
   - 제목: “데이터 흐름”
   - 한 줄 메시지: “수집 → 정제 → DB → 대시보드”
   - 시각자료: 4단계 흐름 다이어그램
   - 각 단계에 아이콘 추가(데이터, 정리, DB, 화면)

5. 핵심 기능 1(시간/요일)
   - 제목: “언제 위험한가”
   - 한 줄 메시지: “시간/요일별 위험 패턴을 바로 비교”
   - 시각자료: 시간/요일 그래프 1~2개 스크린샷
   - 강조: “밤/주말 위험도 상승” 라벨

6. 핵심 기능 2(지역/핫스팟)
   - 제목: “어디가 위험한가”
   - 한 줄 메시지: “지도에서 위험 지역을 즉시 확인”
   - 시각자료: 핫스팟 지도 스크린샷
   - 강조: 위험 지역 마커 확대 컷

7. 투자/확장 포인트
   - 제목: “확장 가능성”
   - 한 줄 메시지: “다른 도시·데이터로 쉽게 확장 가능”
   - 시각자료: 서울 → 다른 도시 화살표 지도
   - 보조 키워드: “유동인구/날씨/행사 데이터 추가 가능”

8. 데모 예고 + 한 줄 결론
   - 제목: “이제 시연”
   - 한 줄 메시지: “지금부터 4분 데모로 직접 보여드립니다”
   - 시각자료: 데모 화면 썸네일 1장
   - 하단 자막: “핵심: 지금 위험한 곳을 바로 보여준다”

### 발표 흐름 요약
- 결론 → 문제 → 해결 → 근거(기능) → 확장 → 시연 예고
